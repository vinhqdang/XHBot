# XHBot: eXplainable Heterophily-aware Graph Neural Networks for Social Bot Detection

This repository contains a **self-contained, reproducible** implementation of XHBot
and all experiments reported in the manuscript. Every number, table, and figure in
the paper is regenerated from the code here — no external datasets or hidden
artifacts are required.

## Why a synthetic benchmark?

The original TwiBot-20 / TwiBot-22 / Cresci-2017 corpora are access-gated and cannot
be redistributed. To keep this repository fully reproducible, the experiments run on
**controlled synthetic heterophily benchmarks** that emulate the camouflage regimes
of those datasets (different scales, bot prevalence, and camouflage degrees). The
benchmarks are deterministic given a seed and are generated by `data_loader.py`. They
are explicitly synthetic emulations, designed to stress-test the relation-camouflage
phenomenon the paper studies; they are not the original corpora.

## Installation

```bash
pip install -r requirements.txt
```

Dependencies: `torch` (CPU is fine), `numpy`, `scikit-learn`, `networkx`,
`matplotlib`.

## Reproducing the results

```bash
# 1. Run all experiments (writes results/*.json). ~30-45 min on CPU.
python run_experiments.py            # add --quick for a fast single-seed smoke test

# 2. Regenerate the figures from those results.
python generate_q1_figures.py        # performance, ablation, robustness
python generate_figures.py           # t-SNE of learned embeddings
python generate_hfe_figure.py        # HFE explanation subgraph
```

Outputs:

| File | Manuscript element |
|------|--------------------|
| `results/main_comparison.json` | Table 1 (XHBot vs. baselines, 3 benchmarks) |
| `results/ablation.json` | Table 2 (component ablation on TwiBot-20) |
| `results/robustness.json` | Robustness figure (F1 vs. camouflage degree) |
| `results/explainability.json` | Table 3 + Section 4.6 case study |
| `manuscript/figures/*.png` | All figures |

All results are averaged over seeds `{0, 1, 2}` (see `SEEDS` in
`run_experiments.py`).

## Code layout

```
data_loader.py         # deterministic synthetic benchmark generator
train.py               # unified training/eval protocol (shared by all models)
run_experiments.py     # orchestrates every experiment -> results/*.json
explain.py             # explainability metrics (Table 3) + HFE case study
model/
  sgtr.py              # Spectral-Guided Topology Refinement
  thca.py              # Tri-Channel Heterophily-Aware Aggregation
  cpd.py               # Contrastive Prototype Disentanglement
  xhbot.py             # full XHBot model (with ablation flags)
  hfe.py               # Hierarchical Forensic Explanation (community/motif)
  baselines.py         # StandardGCN, BotRGCN, RGT, CARE-GNN, NeighborSense, HW-GNN
  layers.py            # shared dense-graph layers
generate_*.py          # figure generation from results/
manuscript/            # LaTeX source, figures, response letter
```

## Notes on the implementation

* Everything is plain PyTorch on dense per-relation adjacencies (no
  `torch_geometric`), which keeps the repository easy to install and fully
  deterministic.
* Baselines are faithful but compact re-implementations of each method's core
  mechanism, trained under an identical protocol (same splits, same features, same
  imbalance handling, validation-selected decision threshold) for a fair comparison.
* The ablation flags in `model/xhbot.py` (`use_sgtr`, `use_homophilic`,
  `use_heterophilic`, `use_self`, `use_gating`, `use_cpd`) reproduce Table 2.
